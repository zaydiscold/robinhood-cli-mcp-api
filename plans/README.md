## Findings considered and rejected (so nobody re-audits them)

- **Raw `brokerage execute` writes bypass the notional caps + strategy classification.** Real and
  confirmed: CLI `brokerage execute` (`cli/src/index.ts:727`) and MCP `robinhood_brokerage_execute`
  (`mcp/src/server.ts:740`) call `executeBrokerageRequest` directly, so `optionsOrderNotional` /
  `checkNotionalCaps` and the "never infer naked exposure" classification (both inside
  `gatedBrokerageWrite`) never run. The env write-gate *does* still apply. Not turned into a plan
  yet **only** because the WIP is actively rearchitecting this exact path (`resolveWriteExecution`);
  a plan written now would collide. Revisit once the WIP lands — the fix is to route `execute`
  writes through `gatedBrokerageWrite` (or lift the cap check into `executeBrokerageRequest`).
- **The three oversized files (`lib.ts` 7.4K, `index.ts` 4.1K, `server.ts` 2.5K).** Real, but an
  L-effort HIGH-risk refactor that breaks both CLI and MCP if botched, and the WIP is mid-flight in
  all three. Not worth handing a weak executor. Owner-led, incremental extraction only.
- **344 lint warnings (mostly `@typescript-eslint/no-explicit-any`).** Real, but the private
  Robinhood API payloads are genuinely `any`-shaped; a blanket typing sweep is M-effort with real
  regression risk and low leverage until focused Zod parsers exist. Defer to a typed-parser plan.
- **Owned-accounts cache not keyed to auth identity (`cli/src/lib.ts:2419`).** Real but low
  likelihood (requires token rotation mid-process without restart). Folded as a follow-up note in
  Plan 002's maintenance section rather than its own plan.
- **`whatif` is a Taylor approximation with no large-move validity bounds.** Real but LOW leverage;
  the math is correct for small moves and the output is labeled a scenario estimate. Nice-to-have
  warning, not a defect. Deferred.
- **`calendar` "earnings" coverage.** Not a finding — the function already emits an explicit
  warning that earnings dates aren't available via the brokerage API; the label is honest.
- **Concentration excludes cash.** Not a finding — by-design portfolio-weight-of-invested and
  honestly labeled "Concentration by Underlying." No false claim.
- **`vite` override 2 minor versions behind; `hono` override may be dead.** Trivial; roll into
  routine dependency maintenance, not an advisor plan. (Confirm `pnpm list hono` — if absent,
  drop the override line in `package.json`.)
- **Sensitive local files at mode 0644 (`trading-log.md`, `local/`).** Real but these are
  git-ignored local artifacts on a single-user machine; a `chmod`/umask note, not an executor code

- **No regenerate-and-diff CI gate for generated api-map artifacts.** Real DX gap; a clean
